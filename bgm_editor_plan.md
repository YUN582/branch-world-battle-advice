# BGM 에디터 & 플레이어 개선 구현 계획

## 목표
BGM 미리듣기 플레이어 UI를 통합·개선하고, 파형 기반 오디오 편집기 + 10MB 자동 압축 기능을 추가하여
"음악 삽입 → 편집 → 압축 → 코코포리아 임포트" 워크플로우를 완성한다.

## 사전 진단 결과

### 관련 파일
- `content/bgm-player.js` — 현재 미리듣기 플레이어 (HTML5 Audio + 시크바)
- `content/redux-injector.js` — BGM URL 조회 + Firestore SDK + **이미지 업로드 코드** (음악에 재활용 가능)
- `content/overlay.js` — Web Audio API 기반 SFX 시스템 (AudioContext/BufferSource/GainNode 구현 참조)
- `content/site-volume-page.js` — 마스터 볼륨 AudioContext 인터셉트

### 기존 인프라
| 항목 | 상태 |
|------|------|
| Firebase Auth 토큰 획득 | ✅ `_getFirebaseAuthToken()` |
| 파일 업로드 엔드포인트 | ✅ `uploadFileV2` Cloud Function |
| SHA256 해시 계산 | ✅ `_computeSha256()` |
| CDN URL 생성 | ✅ `_CDN_BASE + result.name` |
| Firestore userMedia 스키마 | ✅ 17개 필드 문서화됨 |
| 오디오 업로드 기능 | ❌ 아직 없음 (이미지 업로드 패턴 참조) |
| 오디오 코덱/DSP | ❌ 외부 라이브러리 없음 |

### 업로드 아키텍처 (이미지 업로드 기반)
```
1. Firebase Auth 토큰 획득 (IndexedDB에서)
2. Blob → SHA256 해시 계산
3. FormData로 uploadFileV2 Cloud Function에 POST
4. CDN URL 반환: https://storage.ccfolia-cdn.net/users/{uid}/files/{hash}
5. Firestore userMedia 문서 생성
```

### userMedia 엔티티 구조 (BGM용)
```javascript
{
  _id: "문서ID",
  owner: "사용자UID",
  roomId: "방ID",
  dir: "bgm02",           // BGM 카테고리
  name: "표시 이름",
  url: "CDN URL",
  volume: 0.75,
  loop: true,
  order: 0,
  contentType: "audio/mpeg",
  size: 7712098,           // 바이트
  uploaded: true,
  external: false,
  createdAt: Date.now(),
  updatedAt: Date.now()
}
```

### M1: UI 개선 — 통합 플레이어 패널
**목표**: 재생 컨트롤 + 볼륨 + 루프를 하나의 뮤직 플레이어 패널로 통합

#### 현재 레이아웃
```
┌─────────────────────────────────┐
│ name: [입력 필드]                │
├─────────────────────────────────┤
│ ▶ ═══════●═══  0:00 / 2:20     │  ← 재생바 (CE 주입)
├─────────────────────────────────┤
│ 🔊 ═══●═════  0.5    🔁        │  ← 볼륨 (ccfolia 네이티브)
├─────────────────────────────────┤
│     삭제        저장             │
└─────────────────────────────────┘
```

#### 개선 레이아웃
```
┌─────────────────────────────────┐
│ name: [입력 필드]                │
├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
│ ┌─────────────────────────────┐ │
│ │  ‖  ═══════●═══  0:45/2:20 │ │  ← 재생/시크바/시간
```

**테스트**: BGM 편집 팝오버 열었을 때 통합 플레이어가 표시되는지, 볼륨·루프·재생이 모두 작동하는지

### M2: 파형 기반 오디오 편집기
**목표**: 팝오버 내에서 오디오 파형 표시 + 트림/컷 편집

#### UI 구조
```
┌─────────────────────────────────────┐
│ name: [입력 필드]                    │
├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
│ ┌─────────── 편집 모드 ───────────┐ │
│ │ [파형 캔버스 ════█████════════] │ │  ← 파형 + 선택 영역
│ │  ◁──────●──────▷               │ │  ← 트림 핸들 (시작/끝)
│ │  ▶ ━━━━●━━━━  0:45/2:20       │ │  ← 미리듣기
│ │                                 │ │
│ │  [앞 자르기] [뒤 자르기]         │ │
│ │  [구간 제거] [되돌리기]          │ │
│ │  원본: 8.5MB → 편집 후: 6.2MB   │ │  ← 예상 크기
│ └─────────────────────────────────┘ │
│                                     │
│  🔊──●──── 0.5              🔁     │
├─────────────────────────────────────┤
│    [편집 완료]    삭제     저장      │
└─────────────────────────────────────┘
```

#### 기술 구현
- **파형 생성**: `AudioContext.decodeAudioData()` → peak 추출 → Canvas 렌더
- **트림**: 드래그 가능한 시작/끝 마커 (파형 위에 오버레이)
- **구간 제거**: 선택 영역을 빼고 앞뒤를 연결
- **되돌리기**: 편집 히스토리 스택 (undo)
- **미리듣기**: 편집된 영역만 Web Audio API로 재생
- **크기 추정**: `편집 후 길이 / 원본 길이 × 원본 파일 크기`

#### 작업 항목
- [ ] AudioBuffer 디코딩 + 피크 데이터 추출 유틸리티
- [ ] Canvas 기반 파형 렌더러
- [ ] 트림 핸들 (시작/끝 마커 드래그)
- [ ] 구간 선택 & 제거 기능
- [ ] 편집 히스토리 (undo/redo)
- [ ] 편집된 오디오 미리듣기 (Web Audio)
- [ ] 파일 크기 예상 표시

**테스트**: 파형이 정확히 그려지는지, 트림/컷 후 미리듣기가 정상 작동하는지, undo가 동작하는지

### M3: 10MB 자동 압축
**목표**: 편집 완료된 오디오가 10MB를 초과하면 자동으로 품질을 낮추어 10MB 이내로 맞춤

#### 기술 접근
1. **OfflineAudioContext**로 편집된 AudioBuffer를 렌더링
2. **MediaRecorder** (audio/webm;codecs=opus)로 인코딩
   - 브라우저 내장 → 외부 라이브러리 불필요
   - Opus 코덱은 음악 압축에 매우 효율적
3. **이진 탐색**으로 최적 비트레이트 결정:
   - 초기값: `(10MB × 8) / 오디오_길이_초` = 최대 가능 비트레이트
   - 인코딩 → 크기 확인 → 비트레이트 조정 → 반복
   - 10MB 이하이면서 최대한 높은 품질 찾기
4. 결과물 형식: `audio/webm` (Chrome 재생 완벽 지원)

#### 작업 항목
- [ ] AudioBuffer → Blob 변환 유틸리티 (OfflineAudioContext)
- [ ] MediaRecorder 기반 인코딩 (Opus)
- [ ] 이진 탐색 비트레이트 최적화
- [ ] 압축 진행률 UI (프로그레스 바)
- [ ] 압축 전/후 크기 표시

**테스트**: 15MB 오디오 → 자동 압축 → 10MB 이하 확인, 음질이 허용 가능한 수준인지

---

### M4: 업로드 워크플로우 통합
**목표**: 편집+압축 완료된 오디오를 코코포리아에 직접 업로드

#### 워크플로우
```
사용자가 "편집" 클릭
    ↓
에디터 열림 (파형 표시)
    ↓
편집 (트림/컷)
    ↓
"편집 완료" 클릭
    ↓
크기 체크 → >10MB면 자동 압축
    ↓
uploadFileV2로 업로드 (기존 이미지 업로드 패턴 재활용)
    ↓
Firestore userMedia 문서 업데이트
    ↓
코코포리아 BGM 목록에 반영
```

#### 작업 항목
- [ ] 오디오 업로드 함수 (redux-injector.js에 추가 — `_uploadAudioToStorage`)
- [ ] Firestore userMedia 문서 생성/업데이트
- [ ] ISOLATED ↔ MAIN 크로스월드 이벤트 (`bwbr-upload-audio` / `bwbr-upload-audio-result`)
- [ ] 업로드 진행률 UI
- [ ] 에러 처리 (토큰 만료, 네트워크 오류 등)
- [ ] 파일 입력 인터셉트: 새 BGM 추가 시 편집기 자동 연결

**테스트**: 편집→압축→업로드 전체 플로우, 업로드된 BGM이 코코포리아 BGM 목록에 나타나는지

## 영향 범위
- `content/bgm-player.js` — 대폭 리팩토링 (UI 통합 + 편집기 추가)

## 기술 선택 근거
- **Web Audio API**: 브라우저 내장, 디코딩·분석·렌더링 모두 가능
- **MediaRecorder + Opus**: 외부 라이브러리 없이 고효율 압축
- **Canvas 2D**: 파형 렌더링에 충분, WebGL 불필요
- **OfflineAudioContext**: 실시간이 아닌 즉시 렌더링으로 편집 결과물 생성
